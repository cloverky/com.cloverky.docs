# 📱 프리지(Fridge) 플러터플로우 화면 제작 가이드

## 1. 화면 1: 로그인 화면 (Login Screen)
**목적:** 등록된 사용자 식별 및 앱 진입점.

### [UI 구성]
- **배경:** 화이트 톤의 깔끔한 디자인.
- **중앙:** 프리지 로고 및 서비스 슬로건 ("나보다 내 냉장고를 더 잘 아는 AI").
- **하단:** **[Google 로그인 버튼]**.

### [동작 로직 (임시)]
- **버튼 클릭 시:** 실제 인증 로직 없이 무조건 `Success`로 간주하고 **화면 2(대시보드)**로 이동.
- **제약 조건:** 등록된 유저 기반의 DB 연동은 추후 구현 예정 (현재는 화면 흐름만 구축).

---

## 2. 화면 2: 메인 대시보드 (Main Dashboard)
**목적:** 4인방 에이전트(`manager`, `chef`, `reviewer`, `shopper`)의 활동 요약 및 실행.

### [A. 상단: 실시간 재고 현황 (`manager.py` 기반)]
- **위젯:** 리니어 게이지(Linear Gauge) 또는 원형 프로그레스 바.
- **내용:** "현재 냉장고 신선도 85%".
- **알림 배지:** 유통기한 D-1 재료가 있을 경우 우측 상단에 빨간색 느낌표 노출.

### [B. 중앙: 맞춤형 레시피 카드 (`chef.py` 기반)]
- **위젯:** 스와이프 가능한 카드 뷰 (Swipeable Card Stack).
- **내용:**
    - 추천 메뉴 이름 (예: "오늘 저녁은 **마파두부** 어때요?").
    - 활용되는 냉장고 재료 리스트 노출.
- **버튼:** [레시피 보기], [나중에 하기].

### [C. 하단: 에이전트 상태 바 (Floating Navigation Bar)]
- **에이전트별 상태 메시지 스크롤:**
    - **Reviewer:** "어제 먹은 된장찌개 맛은 어땠나요?" (텍스트 슬라이드).
    - **Shopper:** "현재 부족한 재료 2개가 장바구니에 있습니다."
- **플로팅 액션 버튼(FAB):** 카메라 아이콘 (재고 등록/`manager` 호출).

---

## 🛠️ 플러터플로우 구현 가이드 (Instruction)
1. **Auth:** 초기 설정에서 Google Authentication을 활성화하되, 액션(Action) 설정에서 `Navigate to Dashboard`를 직접 연결하여 즉시 로그인이 가능하게 할 것.
2. **State Management:** 냉장고 신선도 수치와 레시피 카드의 텍스트는 `App State` 변수로 설정하여 추후 파이썬 백엔드 API와 연결하기 쉽게 구성할 것.
3. **Theme:** 맑은 민트(Mint)와 화이트(White)를 주 색상으로 사용하여 '신선함'을 강조할 것.
